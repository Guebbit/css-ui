# AI README

When you add or edit a component, always review its documentation page in `/home/runner/work/css-ui/css-ui/docs`.

Checklist:
- make sure the component documentation has a `## Classes` section
- make sure classes are listed and briefly explained
- make sure the component documentation has a `## SCSS variables` section
- in the SCSS variables table, mention both the SCSS variable and the matching CSS variable when available
- if a SCSS variable has no matching CSS variable, write `—`
- if a component only inherits classes or variables, say that clearly
- after the local update, do a repo-wide pass on all component docs to keep the same documentation structure
